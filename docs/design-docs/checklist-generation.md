## 1. `docs/design-docs/checklist-generation.md`

이건 체크리스트 생성 로직 설명 문서.

아래 그대로 복붙하면 돼.

````md
# checklist-generation.md

## 1. 문서 목적

이 문서는 PackUp의 핵심 기능인 “상황 기반 체크리스트 자동 생성” 로직을 정의한다.

본 문서는 다음 내용을 다룬다.

- 상황별 템플릿 구조
- 조건 기반 항목 추가 방식
- 사용자 추가 항목 처리 방식
- 저장 구조와 UI 구조의 연결 방식

---

## 2. 기능 목표

사용자가 특정 상황을 선택하면,
해당 상황에 맞는 준비물 리스트를 자동으로 생성해야 한다.

현재 MVP 목표:
- 여행
- 학교
- 헬스

---

## 3. 생성 방식 개요

체크리스트 생성은 아래 순서로 동작한다.

1. 사용자가 상황을 선택한다.
2. 선택한 상황에 맞는 기본 템플릿을 불러온다.
3. 추가 조건이 있다면 조건에 따라 항목을 추가하거나 제외한다.
4. 사용자 정의 항목이 있다면 기존 리스트와 병합한다.
5. 최종 체크리스트를 상태와 저장소에 반영한다.

---

## 4. 기본 데이터 구조

```ts
type Scenario = "travel" | "school" | "gym";

interface ChecklistItem {
    id: string;
    name: string;
    checked: boolean;
}

interface Checklist {
    id: string;
    scenario: Scenario;
    title: string;
    conditions?: Record<string, string | boolean | number>;
    items: ChecklistItem[];
    createdAt: number;
    updatedAt: number;
}
````

## 6. 조건 기반 확장 방식

MVP에서는 복잡한 추천 시스템 대신 단순 조건 분기만 지원한다.

예시:

### 여행

* 해외 선택 시 → 여권 포함
* 2박 이상 선택 시 → 여벌옷 추가 가능

### 헬스

* 샤워 여부 선택 시 → 세면도구 추가

규칙:

* 조건 로직은 템플릿을 완전히 바꾸지 않는다
* 기본 리스트 + 추가 항목 구조를 유지한다

---

## 7. 사용자 추가 항목 처리

사용자는 자동 생성된 리스트에 직접 항목을 추가할 수 있다.

규칙:

* 사용자 추가 항목도 동일한 item 구조를 따른다
* 자동 생성 항목과 사용자 항목을 구분해야 할 필요가 생기면 이후 `source` 필드를 추가한다
* 현재 MVP에서는 같은 구조로 통합 관리한다

---

## 8. 저장 구조

체크리스트는 여러 개 저장 가능해야 한다.

저장 위치:

* localStorage

저장 키 예시:

* `packup_checklists`

규칙:

* 상태 변경 시 저장
* 앱 초기 진입 시 localStorage에서 복원
* 체크리스트 단위로 저장한다

---

## 9. 예외 상황

### 1. 템플릿이 없는 상황 선택

* 에러 대신 빈 리스트 또는 안내 메시지 표시

### 2. localStorage 데이터 파손

* 기본값으로 초기화

### 3. 동일 이름 항목 추가

* MVP에서는 허용
* 추후 중복 방지 옵션 검토 가능

---

## 10. 설계 원칙

* 단순 템플릿 기반으로 시작한다
* 복잡한 추천 기능은 MVP에 포함하지 않는다
* UI 상태와 저장 구조를 일치시킨다
* 나중에 추천 시스템이 붙어도 구조가 버티도록 만든다

---

## 11. 향후 확장 방향

* 자동 추천 시스템
* 사용자별 템플릿 학습
* 자주 빠뜨리는 항목 분석
* 날씨/일정 기반 동적 추천

