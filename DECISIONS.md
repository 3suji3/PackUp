## 2026-04-22

### 체크리스트 화면 분리

- 문제 상황: 생성된 체크리스트 표시 UI와 상태 변경 로직이 `src/app/page.tsx` 한 파일에 함께 모이기 시작했다.
- 선택지: `page.tsx`에 계속 유지 / 체크리스트 화면을 feature 컴포넌트로 분리
- 선택 이유: 이번 단계 목표가 UI와 로직 분리 시작이므로, 상태는 부모가 유지하고 렌더링은 feature 컴포넌트로 분리하는 방식이 MVP 범위에서 가장 단순하다.
- trade-off: 아직 `currentChecklist` 상태는 페이지 로컬 상태라 다른 화면과 공유되지 않는다. 이후 저장/재사용 단계에서 Context 또는 저장 계층 확장이 필요할 수 있다.

### 체크 상태 갱신 위치

- 문제 상황: 체크/해제 동작을 어디서 처리할지 정해야 했다.
- 선택지: 체크리스트 화면 내부에서 직접 상태 보유 / 부모 페이지가 상태를 보유하고 콜백 전달
- 선택 이유: 단일 상태 소스를 유지하면 `updatedAt` 갱신과 이후 저장 로직 연결이 쉬워진다.
- trade-off: props 전달이 늘어나지만 현재 MVP 규모에서는 복잡도가 낮고 흐름 설명이 쉽다.
